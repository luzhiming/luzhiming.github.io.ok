使用Git遇到的问题以及得出的套路

遇到的问题

场景1：

主干有紧急Bug，但是此时我们的开发是在主干上进行的。

场景2：

主干有紧急功能，但是此时我们的开发是在主干上进行的。

如何解决上述问题:

在没使用分支的情况下

  1. git checkout commitId
  2. git branch –b newBranchName
  3. git push

    在分支使用合理的情况下，上述问题一版不会存在

所以总结如下

每个成模块的功能要使用一个单独的分支。
例如上面的情况或者有A,B,C三个模块需要开发，你开发完了A模块，此时QA测试或者等上线流程。
此时你可以新建立一个分支继续开发B模块，以及C模块，A上线完毕之后，将B以及C merge回主干即可。
每个bug都要使用一个分支,分支名字建议用icafe的bug单号例如[git checkout –b fixbug-1234]，解决完bug再删除分支即可。
开发新功能及修复bug最佳实践
